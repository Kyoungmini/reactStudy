React 이해를 위해 정보 SPA , MPA, SEO

검색 엔직 최적화 SEO(Search Engine Optimization)

  • 검색엔진은 검색 최적화(SEO)를 위해 웹 서버에 있는 HTML 문서를 분석하고 정리(indexing)한다.
  • HTML을 분석 추출하는 행위를 크롤링(crawling) 혹은 스크레이핑(scraping)라 부르고 소프트웨어를 크롤러(crawler)라고 한다.
  • 크롤러의 indexing 기준은 메타,시멘틱 요소, a 태그, pre-rendering 속도 등이 있다.
  • SEO의 대상에는 자바스크립트가 포함되지 않지만 구글은 자바스크립트가 생성한 DOM Tree 크롤링한다.

네이티브 앱 UX를 제공하는 SPA (Single Page Application)

  • 싱글페이지 어플리케이션(SAP)은 한개의 페이지로만 구성된 어플리케이션이다.
  • SPA는 다른 페이지를 요청할 수 없고 자바스크립트로 생성한 DOM을 반환 하는 구조이다
  • 웹어플리케이션에 필요한 모든 리소스(js,css,html,img ... )를 처음 한번 다운 받고 렌더하는 CSR(Client Side Rendering) 구조다.
  • 처음 로딩할 때 다운로드로 시간이 오래 걸리지만 네이티브 앱과 같은 빠르고 자연스러운 페이지 전환 UX(User eXperience)를 제공한다.
  • 브라우저에서 네이트브앱 동작하기 때문에 낯설게 느낄 수 있다.
  • HTML문서가 비어있고 각 페이지별 메타를 작성할 수 없기 때문에 SEO가 굉장히 어렵다.

전통적인 웹 어플리케이션 UX MPA(Multiple Page Application)

  • 멀티페이지 어플리케이션(MPA)은 기존의 웹서버처럼 여러 페이지로 구성된 웹 어플리케이션이다
  • MPA는 요청한 페이지를 다운받아 브라우저에서 출력하는 새로고침이 일어난다.
  • 요청한 페이지는 필요한 리소스를 서버에서 다운로드 받는다.(중복 리소스를 계속 다운로드한다.=>서버 부하)
  • 처음 로딩할 때 속도가 빠르지만 호출되는 페이지를 새로고침으로 렌더하기 때문에 전환이 느리다.(깜빡임)
  • SSR(Server Side Rendering) :서버에서 클라이언트에 출력될 페이지를 구성하고 내보낸다.
  • 오래된 웹 어플리케이션 UX로 친숙하게 사용할 수 있다.
  • MPA는 각 페이지에 sementic, a, meta 태그가 작성되어 SEO가 용이하다.

React란 무엇인가?

  • React는 javascript 라이브러리로 싱글페이지 어플리케이션을 만들기 위해 페이스북에서 개발 되었다.
    • 프레임워크: 어플리케이션을 제어, 재구성, 흐름제어 등 주도권을 가진 형태
    • 라이브러리: 어플리케이션이 필요에 의해 추가한 도구
  • React는 싱글페이지만 호출되도록 흐름을 제어하고 있어 프레임워크라 불리기도 한다.(프레임워크는 무겁고 배우기 힘들다는 이미지가 있어 라이브러리로 불리기를 고집하고 있다.)
  • React는 SPA(Single Page Application)로 CSR(Client Side Rendering)의 특징인 네이티브앱의 UX를 제공한다.
  • React는 SEO 문제를 안고 있지만 pre-rendering인 react-snap과 next, router 라이브러리, Virtual DOM로 어느 정도 해결할 수 있다.
  • React는 Virtual DOM으로 DOM Tree 최적화와 DOM 생명주기를 관리한다.
  • React는 JSX를 제공하고 있어 자바스크립트에서 HTML을 작성하는 경험을 제공한다.
  • React의 js는 독자인 DOM(Virtual DOM) 객체를 제공한다.
  • React의 js는 표준 자바스크립트로 컴파일되어 배포된다.
